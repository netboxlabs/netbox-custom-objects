# GraphQL Support for Dynamic Custom Object Models

This document explains how to use the GraphQL registration system for dynamic custom object models generated by `get_model()`.

## Overview

The GraphQL registration system automatically creates strawberry_django types, filters, and orderings for dynamic models generated by `CustomObjectType.get_model()`. This allows you to query and filter custom objects through GraphQL.

## Components

### 1. CustomObjectGraphQLRegistry

The `CustomObjectGraphQLRegistry` class manages the registration of GraphQL components for dynamic models:

- **Types**: strawberry_django types for querying objects
- **Filters**: strawberry_django filters for filtering objects

### 2. Automatic Registration

GraphQL components are automatically registered when:
- A `CustomObjectType` is created
- A dynamic model is generated via `get_model()`
- A model is created in the database via `create_model()`

### 3. Automatic Cleanup

GraphQL components are automatically unregistered when:
- A `CustomObjectType` is deleted
- The model cache is cleared

## Usage

### Basic Usage

```python
from netbox_custom_objects.models import CustomObjectType

# Create a custom object type
custom_object_type = CustomObjectType.objects.create(
    name="MyObject",
    display_name="My Object",
    slug="my-object"
)

# Add some fields
custom_object_type.fields.create(
    name="name",
    label="Name",
    type="text",
    required=True
)

custom_object_type.fields.create(
    name="count",
    label="Count",
    type="integer"
)

# The GraphQL components are automatically registered
model = custom_object_type.get_model()
# GraphQL type, filter, and ordering are now available
```

### Manual Registration

If you need to manually register GraphQL components:

```python
from netbox_custom_objects.graphql import register_dynamic_model_graphql

# Register GraphQL components for a model
register_dynamic_model_graphql(model, custom_object_type)
```

### Manual Unregistration

If you need to manually unregister GraphQL components:

```python
from netbox_custom_objects.graphql import unregister_dynamic_model_graphql

# Unregister GraphQL components for a model
unregister_dynamic_model_graphql(model.__name__)
```

### Accessing Registered Components

```python
from netbox_custom_objects.graphql import CustomObjectGraphQLRegistry

# Get the GraphQL type for a model
graphql_type = CustomObjectGraphQLRegistry.get_type("MyObjectModel")

# Get the filter for a model
filter_type = CustomObjectGraphQLRegistry.get_filter("MyObjectModel")

```

## Supported Field Types

The GraphQL registration system supports the following field types:

- **text**: String field with text filtering
- **longtext**: String field with text filtering
- **integer**: Integer field with numeric filtering
- **decimal**: Decimal field with numeric filtering
- **boolean**: Boolean field with boolean filtering
- **date**: Date field with date filtering
- **datetime**: DateTime field with datetime filtering
- **url**: URL field with text filtering
- **json**: JSON field with JSON filtering
- **select**: Choice field with string filtering
- **multiselect**: Multiple choice field with list filtering
- **object**: Related object field (resolved to related model type)
- **multiobject**: Multiple related object field (resolved to list of related model types)

## GraphQL Schema Integration

The dynamic GraphQL components are automatically integrated into the NetBox GraphQL schema. You can query custom objects like any other NetBox object:

```graphql
query {
  myObjectModelList {
    id
    name
    count
    created
    lastUpdated
  }
}
```

Filtering is also supported:

```graphql
query {
  myObjectModelList(
    filter: { name: { icontains: "test" } }
  ) {
    id
    name
    count
  }
}
```

## Implementation Details

### Type Generation

The system uses `strawberry_django.type(model, fields='__all__')` to automatically generate types with all model fields.

### Filter Generation

Filters are generated using `strawberry_django.filter_type(model, lookups=True)` with custom field filters added dynamically based on field types.


## Customization

### Custom Field Types

To add support for custom field types, extend the `_get_filter_field_for_custom_field` method in `CustomObjectGraphQLRegistry`.

### Custom Filters

To add custom filters, extend the `_generate_filter` method in `CustomObjectGraphQLRegistry`.


## Troubleshooting

### Common Issues

1. **Import Errors**: Ensure all required dependencies are installed
2. **Type Resolution**: Check that field types are properly mapped
3. **Cache Issues**: Clear the model cache if types don't update

### Debugging

Enable debug logging to see GraphQL registration events:

```python
import logging
logging.getLogger('netbox_custom_objects.graphql').setLevel(logging.DEBUG)
```

## Examples

### Complete Example

```python
from netbox_custom_objects.models import CustomObjectType, CustomObjectTypeField

# Create a custom object type
custom_object_type = CustomObjectType.objects.create(
    name="Server",
    display_name="Server",
    slug="server"
)

# Add fields
CustomObjectTypeField.objects.create(
    custom_object_type=custom_object_type,
    name="hostname",
    label="Hostname",
    type="text",
    required=True
)

CustomObjectTypeField.objects.create(
    custom_object_type=custom_object_type,
    name="cpu_cores",
    label="CPU Cores",
    type="integer"
)

CustomObjectTypeField.objects.create(
    custom_object_type=custom_object_type,
    name="is_active",
    label="Is Active",
    type="boolean",
    default=True
)

# GraphQL components are automatically registered
# You can now query servers via GraphQL
```

### GraphQL Query Examples

```graphql
# List all servers
query {
  serverList {
    id
    hostname
    cpuCores
    isActive
    created
    lastUpdated
  }
}

# Filter servers by hostname
query {
  serverList(filter: { hostname: { icontains: "web" } }) {
    id
    hostname
    cpuCores
  }
}

```

## Integration with NetBox

The GraphQL system integrates seamlessly with NetBox's existing GraphQL infrastructure:

- Uses NetBox's `PrimaryModelFilterMixin` for consistent filtering
- Follows NetBox's GraphQL naming conventions
- Integrates with NetBox's authentication and permissions system
- Supports NetBox's standard field types and lookups
